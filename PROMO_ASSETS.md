# Promo Assets — what to create before launch (Step 6)

Binary assets can't be generated in code — capture these once (30 min total)
and drop into `frontend/public/` or reference from launch posts.

## Screenshots (PNG, phone frame, 1284×2778 or 1170×2532)
| File | Mode | What to show |
|------|------|--------------|
| `shot-swipe.png` | Swipe | A question card mid-swipe, AI explanation peek |
| `shot-bug.png` | Bug Hunting | Highlighted buggy code + "Find the bug" |
| `shot-mock.png` | Mock Interview | Chat with AI evaluation score |
| `shot-stats.png` | Stats | Readiness percentile + streak |

## Demo (15–30s, MP4 or GIF)
- Record: open app → swipe left on a question → AI explanation appears in ~2s
  → swipe right on a known one → percentile grows.
- Tools: CapCut / OBS / macOS Screenshot. Keep < 5 MB for TG/X.
